# Validation — Logistics Graph View

pass condition. Test stubs are written now and fail until implementation lands.

Legend: U = unit (`tests/unit`), I = integration (`tests/integration`), E = e2e
(`tests/e2e`).

## Notes

- AC3–AC5 (R9) are validated and committed before the graph sizing work (R9.6); their
  stub file is independent of any graph component. AC4 is a **regression guard** — it
  already passes today (building recipes are continuous) and must keep passing after the
  MILP change; AC3/AC5 fail until R9 lands.
- The existing `tests/integration/LogisticsSection.test.tsx` (placeholder assertion) is
  replaced by the AC8–AC16 suite.
- Width/layout pure functions (AC17/AC18) are extracted into a testable module so they
  can be unit-tested without React Flow.
- E2E stubs use `test.fixme` until the view exists, then are enabled in the
  implementation loop.
- Integration tests that must mount React Flow may stub/mock `@xyflow/react` where a
  full canvas render is unnecessary (assert on the React tree we control: nodes/edges
  passed in, port markers, link handlers), keeping tests deterministic in jsdom.
